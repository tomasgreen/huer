Usage

Instantiating huer is simple, you cannot however instantiate more than one intance at the time, and if you do the current instance will be replaced by a new instance.

var instance = huer('Your html');

You can choose to instansiate huer by passing HTML or an object as the first (and only) argument.

var instance = huer({
    html: 'Your html',
    showOnInit: false,
    clickOutsideToDismiss: true
});
instance.show();

Basic

Combiding huer with a template engine of sorts is recommended, but of course a simple inline string will do the job.

Custom and default functionality

Creating a simpel wrapper around the script making your own confirmation box is easy. Use vanilla JS, jQuery or whatever you want.

Extended functionality

Using your own html gives you the power to use as many buttons as you want. There's one thing you can think about when constructing your html and that's tabindex on buttons. Not using the tabindex will make the tab-button on your keyboard useless.

Forms

You can also use huer as a form.

Methods

Method Description
void show(function) Shows the view unless already shown. The callback function is optional.
void hide(function) Hides the view unless already hidden. The callback function is optional.
void destroy(function) Destroys the view and sets the instance to null. The callback is optional.
Element getHtml() Returns the content node of the view.
void click(str/el/arr) Add click events on any element in within the container. It also works on touch.
Array query(string) Query elements within the container.
void attr(str/el/arr, attribute, value) Set attributes to elements within the container.

Options

All default options can be changed via huer.globals

huer.globals.blurContent = false;
Property Description
html string/element
(default: 'Your view is empty') 		Set the content of your view, which can be anything you desire.
showOnInit bool
(default: true) 		Turns on/off the automatic display of the view after initialization.
showOnInit bool
(default: true) 		Turns on/off the automatic display of the view after initialization.
preventScroll bool
(default: false) 		Turns on/off scrolling of the document body when the view is active. This feature is a little buggy so I'd suggest not using it...
destroyOnEsc bool
(default: false) 		Turns on/off the dismissal of the view when pressing the ESC-button.
onFeatureFail function(features)
(default: null) 		If the browser lacks the necessary features an alert will be shown. But as an option you could use a callback function to mend the situation. The function takes one argument which will be the feature control object.
onDismiss function()
(default: null) 		Callback for when the view has been dismissed by clicking on the overlay or hitting the ESC-button.
overlayClass string
(default: null) 		Classname appended to the overlay in case you want to change color or add a background image.
blurContent bool
(default: true) 		Turns on/off the blurring of the backdrop. Which elemens are to be blurred can be set with the options blurSelector or blurAllSiblings. This property will be turned off if the browser does not support filter. Observe that filters cannot be animated with the latest stable releases of Firefox and IE (referencedate 2014-11-21). This means that the feature test will pass but not work in those browsers.
blurAllSiblings bool
(default: false) 		Turns on/off blurring all siblings to the container element. This option will superseed the blurSelector.
blurSelector string
(default: [data-huer-effect="blur"]) 		The query that will decide which elements to blur.
wrapContent bool
(default: true) 		Turns on/off wrapping the content in an element with the default content container class.
centerize bool
(default: true) 		Turns on/off placing the body of the view in the veritcal and horizontal center of the window.
